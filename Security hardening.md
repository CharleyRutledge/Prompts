# 🔐 Universal Security Audit Prompt (Vibe-Coded / AI-Generated Apps)

## Role
Act as a **Senior Application Security Engineer** performing a full security audit on this project.

Assume the codebase may have been partially or fully generated by AI and may contain insecure patterns.

---

## Scope

Review the **entire system**, including:

- Frontend code
- Backend services / API routes
- Database schema and queries
- Authentication and authorization logic
- Payment flows and pricing logic
- File handling and storage
- Environment variables and configuration
- Third-party integrations

Assume the application is exposed to **malicious users actively attempting exploitation**.

---

## Output Format

For each vulnerability found, provide:

1. **Vulnerability Type**
2. **Severity** (Critical / High / Medium / Low)
3. **Location** (file + function / endpoint)
4. **How the exploit works (step-by-step)**
5. **Exact fix (code-level guidance)**

---

# 🔎 Core Security Checks

## 1. Secrets Management
- Detect hardcoded API keys, tokens, credentials
- Identify secrets exposed in frontend or public configs
- Ensure sensitive keys are server-side only

---

## 2. Authentication & Authorization
- Identify missing authentication checks
- Detect authorization logic handled only on frontend
- Verify role-based access is enforced server-side

---

## 3. Injection Vulnerabilities
- SQL injection
- NoSQL injection
- Command injection
- Unsanitized inputs

Ensure proper parameterization and validation.

---

## 4. Data Access Control
- Ensure users can only access their own data
- Detect missing or overly broad access rules
- Verify row-level or object-level security enforcement

---

## 5. File Handling
- Unrestricted file uploads
- Executable file upload risks
- Missing validation (type, size, content)

---

## 6. Path Traversal
- Detect unsafe filesystem access using user input
- Ensure directory restrictions are enforced

---

## 7. Sensitive Data Exposure
- Public access to private files or data
- Logs or APIs exposing personal or financial data

---

## 8. Dependency Risks
- Outdated or vulnerable packages
- Unnecessary dependencies increasing attack surface

---

## 9. Debug / Admin Endpoints
- Exposed internal endpoints (e.g. `/admin`, `/debug`, `/export`)
- Missing authentication on sensitive routes

---

## 10. Environment Variable Leakage
- Secrets exposed in:
  - logs
  - error messages
  - API responses
- Server-only variables exposed to client

---

# 💸 Promo Code / Discount / Credit Exploit Checks (Critical)

Audit all logic related to:

- promo codes
- coupons
- discounts
- credits
- referrals
- pricing adjustments

---

## 11. Client-Side Price Manipulation
- Check if pricing is calculated on frontend

**Exploit:**
User modifies request (e.g. sets price to 1)

**Fix:**
All pricing must be calculated and enforced server-side

---

## 12. Unlimited Promo Code Usage
- Missing global or per-user limits

**Exploit:**
Same code reused indefinitely

**Fix:**
- Global usage limits
- Per-user usage limits

---

## 13. Missing User Binding
- Promo not tied to user identity

**Exploit:**
Multiple account creation to reuse “new user” promos

**Fix:**
Bind usage to user identity and apply abuse detection

---

## 14. Race Conditions (Double Spend)
- Non-atomic promo redemption

**Exploit:**
Multiple parallel requests apply same promo

**Fix:**
Use transactions / locking to ensure atomic usage

---

## 15. Negative Pricing / Over-Discounting
- Discounts exceeding total value

**Exploit:**
Final price becomes zero or negative

**Fix:**
- Enforce minimum price = 0
- Cap maximum discount

---

## 16. Promo Stacking Abuse
- Multiple discounts applied together without restriction

**Exploit:**
Stacking leads to excessive discounts

**Fix:**
Restrict stacking or define strict rules

---

## 17. Predictable Promo Codes
- Weak or guessable codes

**Exploit:**
Brute-force or guess valid codes

**Fix:**
Use long, random, unguessable tokens + rate limiting

---

## 18. Missing Expiration Validation
- Expiry not enforced server-side

**Exploit:**
Expired codes still usable

**Fix:**
Validate expiration on backend

---

## 19. Trusting Client Discount Values
- Backend accepts discount values from client

**Exploit:**
User injects arbitrary discount values

**Fix:**
Server must recompute all pricing and ignore client input

---

## 20. Referral Abuse
- Self-referrals or circular referrals allowed

**Exploit:**
Infinite credit generation

**Fix:**
Detect abuse patterns (duplicate identity, device, payment signals)

---

# 🧠 AI-Specific / Vibe Coding Risks

- Missing security logic due to partial generation
- Hallucinated or incomplete validation
- Overly permissive defaults
- Inconsistent patterns across files

---

# 📊 Final Summary

Provide:

1. **Total vulnerabilities found**
2. **Top critical risks**
3. **Promo/discount abuse risks summary**
4. **Most likely real-world attack path (step-by-step)**
5. **Prioritized fix plan before production**

---

## ⚠️ Instructions

Be extremely strict.

Assume:
- Attackers will inspect frontend code
- Attackers will manipulate API requests
- Attackers will automate abuse (scripts, bots)
- Financial logic will be targeted first

Focus on **real exploitability**, not theoretical issues.
